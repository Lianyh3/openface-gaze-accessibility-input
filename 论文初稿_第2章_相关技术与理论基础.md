# 第2章 相关技术与理论基础

本章围绕“基于 OpenFace 的实时视线估计优化”所需的技术背景展开，重点回答三个问题：第一，OpenFace 能提供哪些与视线相关的观测信息，以及这些信息在实时场景下有哪些噪声特征；第二，为什么需要从原始观测走向坐标映射与个体标定；第三，眼控输入从连续坐标到离散事件通常采用哪些交互机制。通过对上述基础进行系统梳理，可为第3章方法设计和第4章系统实现提供统一理论支撑。

## 2.1 OpenFace观测基础

OpenFace 是人脸行为分析领域使用广泛的开源工具链，可输出关键点、头姿、面部动作单元以及 gaze 相关特征[3][4]。在本研究场景中，核心可用信息包括：每帧人脸检测状态、估计置信度、头部姿态（pitch、yaw、roll）以及视线方向相关向量。与专用红外眼动仪相比，OpenFace 的优势在于部署成本低、可复现性强，能够在普通摄像头条件下构建可运行实验链路。

然而，低成本视觉方案的局限同样明确。其一，观测值高度依赖成像质量和人脸可见性，光照变化、遮挡和镜头噪声都会引入不稳定波动。其二，头姿变化会改变眼部外观投影关系，导致同一注视目标在不同姿态下对应观测不一致。其三，跨用户差异显著，包含眼型差异、佩戴眼镜、坐姿习惯等因素。这些问题决定了 OpenFace 输出不能直接等价为“可交互坐标”，而应作为后续优化环节的原始输入。

从时序角度看，实时视线观测同时包含低频趋势与高频扰动。低频趋势反映用户真实注视迁移，高频扰动通常由估计误差、头部微动和帧级抖动构成。若不进行处理，高频扰动会在界面边界处放大为命中跳变，进而传导到事件层引发误触发。由此可见，第3章提出的“归一化—标定—平滑”链路不是附加优化，而是将观测信号转化为交互信号的必要步骤。

## 2.2 视线坐标映射与个体标定基础

**坐标映射问题定义**

在眼控系统中，最终交互对象位于屏幕二维坐标平面，而视觉估计得到的是与相机、头姿和模型假设相关的观测量。二者之间不存在天然一一对应，因此必须建立映射关系。设原始观测为 `(x, y)`，目标屏幕坐标为 `(u, v)`，则映射问题可抽象为函数求解：

`(u, v) = f(x, y, h)`  （2.1）

其中 `h` 表示与头姿、设备布局和用户个体相关的隐含条件。式（2.1）说明，映射并非单纯缩放问题，而是受多因素共同影响。

**归一化与尺度统一**

在工程实践中，首先需要将原始观测变换到统一数值区间，以降低设备尺度差异和输入范围漂移的影响。常见做法是线性归一化：

`x_n = (x - x_min) / (x_max - x_min), y_n = (y - y_min) / (y_max - y_min)`  （2.2）

并结合区间裁剪抑制异常值外溢。归一化本身不解决个体偏差，但可为后续标定提供稳定输入边界，是所有映射策略的前置条件。

**个体标定与仿射模型**

用户之间的系统性偏差往往无法通过统一参数消除，因此眼控场景通常需要个体标定。标定过程通过“已知屏幕点—对应观测值”样本对估计映射参数。考虑本科课题对可解释性和实时性的要求，二维仿射模型是一种较稳妥选择：

`[u, v]^T = A [x_n, y_n, 1]^T`  （2.3）

其中 `A` 为 `2x3` 参数矩阵。与高复杂度非线性模型相比，仿射模型参数少、求解快、可复核性强，且在有限样本条件下不易出现过拟合失控。实际部署中可采用九点标定等策略覆盖中心与边缘区域，以提升全屏映射稳定性。

**标定误差与更新机制**

标定并非一次完成后永久有效。摄像头位置偏移、用户坐姿变化、环境亮度变化都可能使既有参数失效。因而，映射系统通常采用“会话复用 + 条件重标定”的策略：在短时间会话内复用参数以控制交互负担，在命中偏差持续增大时触发重标定以恢复准确性。该策略本质上是效率与精度的动态折中，也是后续系统可靠性设计的重要前提。

## 2.3 时序平滑理论与抖动抑制

**眼动信号的时序噪声特性**

实时视线轨迹可视为“真实注视轨迹 + 观测噪声”的叠加。若直接使用逐帧观测做事件判定，短时噪声会在边界区域诱发高频切换，导致触发不稳定。时序滤波的目标不是改变用户意图，而是在尽量保留注视趋势的同时抑制随机波动。

**指数移动平均（EMA）**

EMA 是最常用的低开销滤波方式之一，其递推形式为：

`p_t = alpha z_t + (1 - alpha) p_(t-1)`  （2.4）

其中 `z_t` 为当前观测，`p_t` 为滤波输出。`alpha` 越小，平滑越强但响应越慢；`alpha` 越大，跟手性更好但抗抖能力下降。EMA 适合在实时系统中作为默认基线，因为其实现简单、计算成本稳定、参数语义直观。

**One-Euro滤波与自适应思想**

1€ Filter（One-Euro）通过速度估计自适应调整截止频率：慢速阶段增强平滑，快速阶段降低平滑强度，以减少拖尾[15]。这种机制尤其适用于“稳定注视 + 快速切换”并存场景。其代价是参数耦合更复杂，需同时考虑 `min_cutoff`、`beta` 与 `d_cutoff`，调参成本高于 EMA。

**平滑与延迟权衡**

在交互系统中，滤波效果不能仅凭“抖动下降”评价，还需同时考察响应延迟。过强平滑可能造成目标切换滞后，降低输入效率；过弱平滑会提高误触发概率。该权衡关系决定了滤波参数必须与触发阈值联合设计，而不能独立调优。第5章将通过任务级指标验证不同参数区间对准确性和效率的综合影响。

## 2.4 停留触发与眼控输入机制

**从连续坐标到离散事件**

眼控输入的核心难点在于：视线是连续信号，而输入行为需要离散事件。若“看见即触发”，系统会将自然扫视误判为操作；若触发条件过严，又会导致输入迟缓。为解决该矛盾，多数眼控系统采用“命中判定 + 停留触发”机制[3][5]。

命中判定负责回答“当前看向哪个目标”，停留触发负责回答“是否形成有效操作”。只有两者同时成立，系统才生成按键或候选选择事件。这种双层判据可显著降低误操作，是无障碍眼控输入中的基础机制。

**停留阈值与状态机建模**

停留机制通常以阈值 `tau` 控制触发时机。当目标连续保持并超过阈值时触发一次事件；目标变化后重置计时。其逻辑可抽象为：

`trigger = 1, if g_t = g_(t-1) and (t - t_s) >= tau and e = 0`  （2.5）

其中 `g_t` 为当前目标，`t_s` 为该目标开始时刻，`e` 为触发抑制标记。式（2.5）体现了三个必要条件：目标一致性、时间累积性、单次触发约束。该建模方式能够有效避免长时间凝视导致的重复输入。

**文本状态管理与候选增强**

实际文本输入除“按键触发”外，还涉及组合区、提交区和候选词等状态管理问题。若系统只统计最终提交文本，容易把“已输入但未提交”误判为识别失败。因而，眼控输入研究通常需要引入状态可观测设计，以区分识别误差、触发误差和流程误差三类问题[18]。

候选机制属于效率增强层，能够减少逐字操作次数，但其有效性依赖底层命中与触发稳定性。换言之，候选并不能替代视线估计优化，而是在优化信号可靠后进一步提升任务完成效率。

## 2.5 本章小结

本章从工具链、映射理论、时序滤波和交互机制四个方面梳理了本文研究所需基础。OpenFace 提供了可复现的低成本观测入口，但其输出在实时场景下存在噪声和个体差异；坐标映射与个体标定用于建立“观测—屏幕”对应关系；时序滤波用于控制抖动与延迟权衡；停留触发与文本状态管理用于将连续视线转化为可控输入事件。上述基础共同构成第3章方法设计与第4章系统实现的理论前提。
