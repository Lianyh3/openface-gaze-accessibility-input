# 第6章 结论与展望

## 6.1 全文工作总结

本文围绕“基于 OpenFace 的实时视线估计优化”开展研究，并以无障碍眼控输入系统作为验证平台，完成了从方法到系统再到实验的闭环。

在方法层，本文将问题抽象为“归一化—标定—平滑”的分层链路，针对实时场景中常见的抖动、漂移和个体偏差给出可解释处理路径。相较于仅依赖原始视觉输出，该链路强调把观测信号转换为可交互信号。

在系统层，本文完成了从连续坐标到离散事件的交互落地，构建了“命中判定 + 停留触发 + 文本状态管理 + 候选机制”的输入流程，并实现会话级日志记录，使实验结果具备可追溯性。

在验证层，本文按“基础链路、闭环任务、一致性验证、失败案例”四类实验组织证据。结果表明：系统能够完成固定文本输入任务，且跨后端输出语义一致；同时，失败案例可被分层定位并解释，不再停留于笼统的准确率描述。

## 6.2 主要结论

结合全文工作，本文形成以下结论。

第一，OpenFace 在静态条件下可提供完整观测，但在实时场景中存在明显质量波动。该现象说明低成本视觉方案若要支持输入任务，必须引入面向交互的后处理链路。

第二，归一化、个体标定和时序平滑的组合能够显著提升信号可用性。该结论不强调复杂模型堆叠，而强调“可解释、可复现、可调参”的工程路径，这与本科毕业设计的实现与验证目标相匹配。

第三，眼控输入性能是多模块耦合结果。即使坐标质量达到可用水平，若提交机制缺失或触发策略不当，任务仍可能失败。因此，系统评估必须覆盖“感知—触发—提交”全链路，而不是只报告单一误差指标。

第四，Python 与 C++ 后端在关键流程上可实现一致行为语义。该结论为后续性能优化和工程迁移提供了稳定接口基础，降低了架构演进中的实现风险。

## 6.3 研究不足与未来展望

本文仍存在以下不足。

（1）样本规模与任务规模有限，当前实验主要面向短句输入和小规模会话，统计稳健性仍需增强。  
（2）被试多样性不足，尚未系统覆盖不同年龄、视力状态和使用习惯下的差异。  
（3）交互反馈机制仍偏基础，对“输入完成但未提交”类失败的预防能力有限。  
（4）当前评价以工程可行性为主，尚未形成大样本条件下的显著性检验。

基于上述不足，后续工作可沿四个方向推进。

第一，扩展多被试、多场景长时实验，建立更完整的数据分层统计。  
第二，完善交互层反馈设计，如提交确认提示、状态可视化与错误恢复引导。  
第三，开展参数敏感性与个体自适应研究，形成“阈值—平滑—任务类型”联合调参策略。  
第四，在保持行为一致性的前提下继续优化运行时性能，探索同进程绑定与更低时延实现。

总体而言，本文工作证明了“普通摄像头 + 开源视觉工具 + 可解释优化链路”在无障碍眼控输入场景中的工程可行性，为后续深入研究提供了可复用的实现基础与实验框架。
