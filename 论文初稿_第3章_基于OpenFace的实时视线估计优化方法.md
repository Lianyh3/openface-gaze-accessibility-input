# 第3章 基于OpenFace的实时视线估计优化方法

## 3.1 优化问题定义与目标

**输入信号与任务定义**

本文以 OpenFace 输出的实时视线观测为基础输入。对每一帧数据，可抽象为三元组：时间戳 `t`、横向视线值 `x`、纵向视线值 `y`。在工程场景中，这组三元数据并不能直接用于眼控交互，其原因在于原始视线值与屏幕交互坐标之间并不存在天然一致关系，且观测信号受到头部微动、光照变化和估计噪声影响，容易产生瞬时偏移。

因此，本文将“实时视线估计优化”定义为以下任务：在不改变基础感知来源的前提下，对原始视线观测序列进行坐标归一化、个体标定与时序平滑，使输出信号满足后续交互模块对稳定性、可映射性和实时性的要求。

**Baseline 在实时场景中的主要问题**

将 OpenFace 原始输出直接用于交互时，通常出现以下问题。

第一，抖动明显。即使用户保持相对静止注视，输出坐标仍会在局部区域持续波动，导致目标命中不稳定。  
第二，个体偏差显著。不同用户的头姿习惯、相机摆位和眼部特征存在差异，统一参数难以保证跨用户一致性。  
第三，误触发风险较高。若不进行平滑与触发控制，短时噪声会被误判为有效注视移动，影响输入可靠性。  
第四，实时链路敏感。优化环节若设计不当，虽然可提升稳定性，但可能引入额外延迟，削弱交互响应。

上述问题说明，本研究重点不在“重新实现视线估计器”，而在“对 baseline 输出进行面向实时应用的可解释优化”。

**优化目标**

结合任务定义与实际需求，本文设定三类优化目标。

（1）稳定性目标：降低静止注视条件下的坐标波动幅度，提升时序一致性。  
（2）可映射目标：通过个体标定修正空间偏差，使视线坐标更准确地对应屏幕目标区域。  
（3）实时性目标：在逐帧处理框架下控制计算开销，保证交互延迟处于可接受范围。

基于上述目标，本文提出“归一化—标定—平滑”的分层优化链路，并在下一节开始逐步展开。

## 3.2 坐标归一化与边界约束

**归一化的必要性**

OpenFace 输出的视线值通常位于相机相关或模型相关坐标范围，而交互目标位于屏幕归一化平面。若直接使用原始值进行区域命中，结果会受到设备位置与尺度变化影响，难以复现。为此，本文首先将原始坐标映射到统一数值区间，为后续标定与平滑提供一致输入。

**线性归一化模型**

设原始横向与纵向坐标分别为 `x`、`y`，其观测区间上下界为 `x_min, x_max` 与 `y_min, y_max`，则归一化结果如式（3.1）所示：

`x_n = (x - x_min) / (x_max - x_min),  y_n = (y - y_min) / (y_max - y_min)`  （3.1）

线性归一化的优点在于简单、可解释、计算开销低，适合实时逐帧处理。

**边界约束与异常处理**

实时采集过程中常出现短时异常值，例如突发遮挡、姿态剧烈变化等。若异常值不受约束，后续标定和平滑阶段会被干扰。本文采用边界裁剪策略，如式（3.2）所示：

`x_n = clip(x_n, 0, 1),  y_n = clip(y_n, 0, 1)`  （3.2）

其中 `clip` 表示将数值限制在给定区间内。该步骤虽然简单，但在实时链路中作用明确：一方面抑制离群点扩散，另一方面保持后续模块输入范围稳定。

本节完成了从“原始观测坐标”到“统一归一化坐标”的转换，并通过边界约束增强了输入鲁棒性。该过程不直接提升精度上限，但能够有效降低后续优化的不确定性，是整个方法链路的基础层。

## 3.3 个体标定建模与参数求解

**个体标定任务设计**

归一化只能统一尺度，无法消除个体差异带来的系统性偏差。为此，本文引入个体标定机制。标定阶段引导用户依次注视预设屏幕点，记录对应的归一化观测坐标与屏幕参考坐标，形成监督样本对。

在本科工程实践条件下，九点标定兼顾实现难度与有效性：采样点覆盖屏幕中心与边缘区域，能够反映全局映射偏差；采样成本可控，便于重复采集与参数更新。

**仿射映射模型**

本文采用二维仿射映射建模归一化坐标到屏幕坐标的关系。设归一化坐标为 `(x_n, y_n)`，标定后坐标为 `(u, v)`，可表示为式（3.3）：

`[u, v]^T = A * [x_n, y_n, 1]^T`  （3.3）

其中 `A` 为 `2x3` 参数矩阵。仿射模型能够表达平移、缩放与旋转等线性几何变换，具有参数少、可解释、实时可用的特点，适合本研究场景。

**参数拟合与稳定项**

在样本矩阵形式下，可通过最小二乘拟合参数 `A`。考虑采样噪声与样本数量有限问题，本文采用带稳定项的优化形式，如式（3.4）所示：

`A* = argmin_A ||XA - Y||^2 + lambda||A||^2`  （3.4）

其中，`X` 为输入样本矩阵，`Y` 为目标坐标矩阵，`lambda` 为稳定项权重。引入稳定项后，参数在面对局部噪声时不易过拟合，有助于提升实时使用阶段的泛化稳定性。

从误差来源看，标定残差主要由三部分组成：其一，用户注视行为本身存在微小漂移；其二，原始观测受姿态变化与噪声影响；其三，仿射模型对非线性畸变的表达能力有限。本文不试图在本科阶段引入高复杂度非线性模型，而是通过稳定项和重复采样策略控制误差上界，使模型保持“可解释、可复现、可实时”。

为便于后续实验解释，本文将标定拟合结果分解为 `mae_x`、`mae_y`、`rmse` 与 `max_abs_error` 四类统计量。该做法能够区分“整体平均误差”与“局部极值误差”，避免仅用单一均值掩盖边缘点偏差。

**标定参数复用与更新策略**

标定参数并非一次性固定。实际使用中，摄像头位置变化、用户坐姿变化都可能导致映射偏差回升。本文采用“可复用 + 可更新”的策略：在会话内复用当前标定参数，在环境变化较大时触发重新标定。该策略兼顾了交互效率与校准准确性。

具体而言，本文将参数生命周期划分为“初始化—会话复用—条件失效—重新拟合”四个阶段。当出现长时间命中偏移、头姿范围显著变化或设备位置调整时，系统触发重新标定。该策略的意义在于避免“每次启动都重新标定”导致的使用负担，同时抑制“长期复用旧参数”导致的精度衰减。

本节通过个体标定将“统一坐标”进一步转化为“用户可用坐标”，解决了跨用户与跨场景偏差问题。相较仅靠归一化处理，标定机制对命中准确性提升更直接，是优化链路中的关键环节。

## 3.4 平滑策略与参数权衡

**噪声特性与平滑需求**

视线观测在时间维度上具有高频波动特征。即使空间映射已经通过标定修正，帧间坐标仍可能出现小幅抖动。对于实时交互而言，这类波动会在目标边界附近放大为命中不稳定问题，因此需要在保持响应速度的前提下进行时序平滑。

**EMA 平滑策略**

指数移动平均（EMA）使用当前观测与历史状态加权融合，计算形式如式（3.5）所示：

`p_t = alpha * z_t + (1 - alpha) * p_(t-1)`  （3.5）

其中 `z_t` 为当前观测，`p_t` 为平滑后输出，`alpha` 为平滑系数。`alpha` 较小时平滑更强但响应较慢，`alpha` 较大时响应更快但抖动抑制较弱。EMA 优点是实现简洁、计算代价低，适合实时链路默认方案。

**One-Euro 平滑策略**

One-Euro 方法根据信号变化速度自适应调整截止频率：慢速阶段增强平滑以减少抖动，快速阶段降低平滑强度以减少拖尾。该方法在“稳定注视”和“快速视线切换”并存场景中具有较好平衡能力。

从参数含义看，`min_cutoff` 决定基础平滑强度，`beta` 决定对速度变化的敏感程度，`d_cutoff` 影响速度估计的平滑程度。该方法参数含义明确，但调参复杂度高于 EMA。

**平滑强度与响应延迟权衡**

平滑并非越强越好。过强平滑虽然能显著降低抖动，但会带来响应滞后，影响目标切换效率；平滑不足则容易误触发。本文将该问题视为“稳定性与响应性”的权衡，并在实验章节通过对比不同平滑模式量化其影响。

从系统角度看，平滑参数的作用可理解为对观测噪声与控制滞后的再分配：更高平滑强度将随机波动转化为确定性延迟，更低平滑强度则将延迟风险转化为触发波动风险。本文采用可切换平滑策略而非单一固定方案，目的在于使不同场景能够依据任务类型选择参数。例如，文本输入场景通常更重视稳定性，目标切换频繁场景更重视响应性。

本节建立了时序平滑机制，核心目标是在低延迟约束下抑制帧间抖动。通过引入 EMA 与 One-Euro 两类策略，方法链路能够根据场景需求在“跟手性”和“稳定性”之间进行可控调整。

## 3.5 实时优化链路与复杂度分析

**链路集成思路**

综合前述方法，本文将实时优化过程组织为逐帧流水：先完成坐标归一化和边界约束，再进行个体标定修正，最后执行时序平滑，输出可供交互模块消费的稳定坐标。该链路按功能分层，便于独立调参与消融分析。

**伪代码描述**

算法3-1给出了实时视线估计优化流程。

```text
输入：原始视线序列 S = {(t_i, x_i, y_i)}
输出：优化后视线序列 P = {(t_i, u_i, v_i)}

for 每一帧观测 (t_i, x_i, y_i):
    1) 归一化坐标并执行边界约束
    2) 若存在标定参数，则执行仿射映射修正
    3) 根据选定策略执行时序平滑
    4) 输出当前帧优化坐标 (u_i, v_i)
end for
```

伪代码强调“方法步骤”而非实现细节，有利于突出论文的逻辑主线与可复现性。

**复杂度与实时可行性**

本方法每帧仅包含常数级坐标变换和低阶状态更新，不依赖高复杂度迭代求解。标定参数拟合发生在标定阶段，不进入每帧主路径；在线阶段主要执行矩阵乘法与滤波更新，因此理论上满足实时处理要求。

在工程解释上，可将计算负担分为“低频任务”和“高频任务”：标定属于低频更新，平滑与映射属于高频逐帧更新。该划分有助于在保证稳定性的同时控制端到端延迟。

需要指出的是，链路实时性不仅由算法复杂度决定，还受数据组织方式影响。若在主路径中引入不必要的格式转换、重复计算或阻塞式操作，即使算法本身复杂度较低，仍会造成端到端延迟抖动。因此，本文在流程设计上强调“主路径最短化”：主路径仅保留归一化、标定映射和平滑更新等必要步骤，其余任务下放至低频侧完成。

**模块可配置性**

为支持不同实验条件，链路允许按需启用或关闭特定模块，例如关闭平滑作为对照组，或切换不同平滑策略进行参数敏感性分析。该可配置性直接服务于第5章实验设计，有助于清晰评估各优化环节的独立贡献。

模块可配置性的另一个作用是增强结论可解释性。通过在同一输入数据上逐项启停模块，可以将性能变化归因到具体环节，避免“多模块同时变化”导致的结论混淆。该思路在第5章的一致性与失败案例分析中将进一步体现。

本节将各优化环节整合为统一实时流程，并说明了其复杂度特征与可配置能力。由此，本文的方法不仅具备理论可解释性，也具备工程可验证性。

## 3.6 本章小结

本章围绕“基于 OpenFace 的实时视线估计优化”给出了完整方法框架。首先，将原始视线观测抽象为可处理的时序坐标问题，并明确稳定性、可映射性与实时性三类目标；其次，通过归一化与边界约束统一输入范围，通过个体标定修正空间偏差，通过时序平滑抑制帧间抖动；最后，将各环节组织为逐帧优化链路，并从复杂度与可配置性角度说明其实时可行性。

本章重点在于提出可解释、可复现的优化方法，不依赖特定实现语言。下一章将在此方法基础上介绍无障碍眼控输入系统的设计与实现，展示优化信号在真实交互链路中的落地过程；第5章进一步通过对比与消融实验验证本章方法的实际效果。
