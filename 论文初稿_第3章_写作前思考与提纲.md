## 3.2 公式与符号清单（写作时必须统一）

### 3.2.1 归一化

设原始横纵坐标为 `x, y`，归一化后为 `x_n, y_n`：

`x_n = (x - x_min) / (x_max - x_min)`  
`y_n = (y - y_min) / (y_max - y_min)`

并执行区间裁剪：

`x_n = clip(x_n, 0, 1)`  
`y_n = clip(y_n, 0, 1)`

### 3.2.2 仿射标定

`[u, v]^T = A * [x_n, y_n, 1]^T`

其中 `A` 为 `2x3` 标定参数矩阵，通过标定样本最小化误差求解。可给出正则化形式：

`A* = argmin_A ||XA - Y||^2 + lambda||A||^2`

### 3.2.3 EMA 平滑

`p_t = alpha * z_t + (1 - alpha) * p_(t-1)`

其中 `z_t` 为当前观测，`p_t` 为平滑结果，`alpha` 控制响应与平滑程度。

### 3.2.4 One-Euro 平滑

给出概念性表达即可：根据速度自适应截止频率，慢速更稳、快速更跟手。  
避免写过长推导，重点说明参数 `min_cutoff, beta, d_cutoff` 的作用。

## 3.4 伪代码表达建议（替代源码描述）

本章建议使用“算法框”而非源码片段。推荐放入 1 段总流程伪代码，名称可写为“算法3-1 实时视线估计优化流程”。

示例写法（可直接改写进正文）：

```text
输入：原始视线序列 S = {(t_i, x_i, y_i)}
输出：优化后视线序列 P = {(t_i, u_i, v_i)}

for 每一帧观测 (t_i, x_i, y_i):
    1) 执行坐标归一化，得到 (x_n, y_n)
    2) 执行边界约束，抑制异常点
    3) 若已完成个体标定，则进行仿射映射修正
    4) 采用指定平滑策略（none / EMA / One-Euro）更新当前坐标
    5) 输出优化后坐标 (u_i, v_i)
end for
```

- 伪代码只写“方法步骤”，不写函数名、文件名、参数解析细节；
