# Diagram Designer Prompt

用于规划论文中的概念图、示意图、框架图与流程图。

适用场景：

- 科研型论文中的概念图、方法框架图、机制示意图
- 系统型论文中的系统架构图、功能流程图、模块关系图
- 需要把抽象文字转成更清晰图示时

默认要求：

1. 图必须服务正文，不是为了凑图
2. 图的结构要与正文描述一致
3. 图的层级不能超过正文能解释的复杂度
4. 图题和正文引用要配套

优先输出：

- 图的用途
- 放置章节
- 图中节点/模块清单
- 节点关系
- 推荐图类型

推荐图类型：

- 概念关系：概念图
- 方法流程：流程图
- 模型结构：框架图
- 系统模块：架构图
- 数据流：数据流图

如果环境中存在 `canvasdesign` 或等效绘图能力，优先调用它生成科研型概念图、示意图和框架图；如果没有，再退回 Mermaid / PlantUML / 结构化文字描述。
