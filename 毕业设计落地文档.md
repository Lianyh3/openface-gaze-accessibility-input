## 1. 研究定位与目标

本课题目标不是只做一个可运行 Demo，而是完成：

1. 算法层：基于 OpenFace 的实时视线估计优化。
2. 系统层：落地一个可用的无障碍眼控输入系统。
3. 评估层：以量化实验证明优化有效（相对 OpenFace 原生 baseline）。

验收关键词：可复现、可量化、可演示。

## 2. 技术路线定稿`

### 2.1 必选项

1. OpenFace（用于关键点、头部姿态、baseline gaze）。
2. 自研视线估计优化模块（几何重建 + 时序稳定 + 个体标定）。
3. 实时眼控输入法前端（注视选择 + 文本输入）。

### 2.2 AI增强项取舍

结论（2026-03-09 更新）：**主线改为直连 OpenAI Responses API；LangChain 降为可选封装；Coze 不进入首版（扩展可选）**。

原因：

1. 论文主贡献应聚焦在“视线估计优化”，AI 增强应尽量减少中间层变量。
2. 直连 OpenAI API 依赖更少、链路更短，排障和复现实验更容易。
3. LangChain 不是过时技术，但更适合多模型/多工具编排场景；当前 MVP 阶段收益不高。
4. Coze 仍作为工程扩展，展示“眼控 + 智能体指令面板”的产品化潜力。

## 4. 系统功能边界

### 4.1 核心功能（必须完成）

1. 摄像头实时采集与人脸关键点追踪。
2. 头部姿态估计与坐标变换。
3. 自研 gaze 向量计算与稳定化输出。
4. 屏幕标定（9点或16点）。
5. 眼控虚拟键盘：注视停留触发（dwell）、删除、空格、确认。
6. 结果可视化：视线方向线、当前注视键高亮、文本输出框。

### 4.2 增强功能（建议完成）

1. OpenAI Responses API 候选词重排与短语补全（LangChain 仅做可选封装）。
2. 简易误触抑制机制（如最小停留阈值 + 去抖）。
3. 用户参数配置界面（阈值、滤波强度、字体大小）。

### 4.3 扩展功能（可选）

1. Coze 智能体指令面板（打开应用、网页搜索、发送固定消息）。
2. 多场景演示脚本（办公输入/聊天输入/系统控制）。

## 6. 实验评估最低标准（毕业线）

至少完成以下实验：

1. 准确性对比：baseline 与优化方法角误差对比（均值/方差）。
2. 实时性对比：FPS 与端到端延迟。
3. 输入性能：固定文本录入任务的 CER 与 WPM/CPM。
4. 稳定性对比：视线抖动标准差（静止注视任务）。
5. 消融实验：证明每个优化模块有独立贡献。

## 7. 答辩演示最低标准

4. 对比 baseline 与优化后体验差异（速度或稳定性）。

## 8. 风险与预案

1. OpenFace 跟踪不稳：
   预案：加光照约束、ROI 限制、失败帧插值。
2. 准确率不足：
   预案：增加个体标定样本、引入非线性映射修正。
3. 实时性不足：
   预案：降低检测频率、异步流水线、裁剪输入分辨率。
4. AI增强收益不明显：
   预案：将 AI 模块降级为可选，先确保核心眼控指标达标。

6. 支持从注视目标序列（`target_id`）经 dwell 判定触发键盘事件（回放已验证）。
