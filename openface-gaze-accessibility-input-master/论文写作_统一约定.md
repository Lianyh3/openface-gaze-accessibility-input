# 论文写作统一约定（合并版）

更新时间：2026-04-04  
来源：由 `论文写作_RULES.md` 与 `论文写作_对话约定.md` 合并精简而来。

适用范围：`论文初稿_*.md` 与后续 Word 定稿整理。

## 1. 主线与定位

1. 全文主线统一为“实时视线估计优化”，眼控系统定位为验证平台。
2. 当前优先完成论文定稿，代码重构与平台迁移作为后续任务。
3. 论证口径统一为“问题 - 方法 - 结果/作用 - 局限”，避免功能清单式叙述。

## 2. 章节结构约定

1. 第1章固定结构：
   - `1.1 研究意义`
   - `1.2 研究内容`
   - `1.3 研究方法`
   - `1.4 论文结构安排`
   - `1.5 本章小结`
2. 目录最多到三级标题，四级内容改用段内编号 `（1）（2）`。
3. 每章必须有本章小结，且小结不重复摘要表述。
4. 第5章建议按五段式组织：
   - `5.1 实验目标、条件与评价口径`
   - `5.2 基础链路与闭环任务结果`
   - `5.3 重复实验稳定性与消融分析`
   - `5.4 失败案例与有效性讨论`
   - `5.5 本章小结`

## 3. 字数分配约定

1. 不追求各章平均分配。
2. 第3/4/5章允许明显更厚，第1/2/6章相对精炼。
3. 建议占比：
   - 第1章：10%~15%
   - 第2章：15%~20%
   - 第3章：25%~30%
   - 第4章：20%~25%
   - 第5章：20%~25%
   - 第6章：8%~12%

## 4. 技术叙述边界

1. 不把“后端切换与回退机制”作为正文主贡献。
2. 不把“Python/C++ 一致性对比”作为核心结论主线。
3. 第4章写“单实现主路径 + 异常检测与恢复机制”。
4. Windows 迁移与重构计划放第6章展望，不写成当前实现事实。

## 5. 公式、编号、文献

1. 公式按章编号：`（3.1）`、`（5.2）`。
2. 正文引用统一写法：`由式（3.1）可见`。
3. 图表、公式、引用编号都必须连续且不重复。
4. 参考文献不少于15篇，英文不少于3篇。
5. 占位引用必须在定稿前替换为真实可查文献条目。

## 6. 实验与结果表达

1. 指标至少覆盖：准确性、稳定性、实时性、输入任务效果。
2. 同时保留成功样例与失败样例分析。
3. 失败分析按层归因（感知层/触发层/提交层）。
4. 比较实验必须说明口径一致（输入、参数、阈值、评价标准）。

## 7. 表达与排版预检查

1. 避免口语化表达（如“我觉得”“很强”）。
2. 不夸大结论，样本有限时明确适用边界。
3. 插图与表格必须先在正文引用再出现。
4. 目录与正文标题必须同步更新。

## 8. 插图规范

1. 第3/4/5章插图保留 `drawio` 可编辑源文件。
2. 图号按“图x-y”命名并与正文引用一致。
3. 风格统一为打印友好（浅底、深色描边、避免高饱和）。
4. 禁止文本压线、压节点、压箭头；信息过多优先换行。
5. 线型语义统一：
   - 实线：主流程
   - 虚线：反馈/辅助路径
6. 每次改图后检查图号、正文引用映射和可读性。

## 9. 提交前清单

1. 目录与正文标题一致。
2. 图表编号、公式编号、引用编号连续。
3. 第1章问题定义与第5章结论口径一致。
4. 参考文献与正文引用一一对应。
5. 满足学校字数与格式要求后再导入模板定稿。
