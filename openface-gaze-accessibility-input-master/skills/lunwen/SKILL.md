# Lunwen

## Overview

这个技能用于把“真实项目事实 + 用户给定模板 + 往届样文 + 文献约束 + 图表截图 + Word 成稿要求”稳定转化为一篇可交付的中文论文。目标不是把论文写厚，而是优先吃透用户给定的模板和样文，再按其真实体量、样式、分页和图表节奏输出 `.docx` 成稿。

## Core Flow

### 1. 锁定输入

这是一个硬闸门步骤。对于学校论文、毕业设计论文、课程设计论文：

- 如果用户尚未提供学校模板、历届样文、开题报告、任务书中的任意一种，默认不应先输出论文设计稿或章节目录
- 首次响应应优先索取这些材料的本地路径，而不是先给通用目录
- 只有在用户明确表示“没有模板/样文，也先按通用格式写”时，才允许退回默认目录与默认版式
- 如果已经拿到模板或样文，才进入目录设计与样式确认阶段

换言之，默认顺序应为：

1. 先要模板/样文/任务书
2. 再分析模板和样文
3. 再回传目录与样式设计
4. 最后才开写正文

不允许把 `3` 提前到 `1` 前面。

先确认以下输入并确定优先级：

1. 学校模板
2. 用户上传的往届样文
3. 用户口头要求
4. 技能默认规则

首次响应论文请求时，必须主动告诉用户可以直接提供模板、历届样文、开题报告、任务书或封面要求的本地路径，示例格式如：

- `D:\论文模板.docx`
- `D:\论文模板.doc`
- `D:\论文样文1.docx`
- `D:\开题报告.pdf`

如果用户给的是 `.doc` 文件，必须先转换成 `.docx` 再进入后续分析；不允许跳过格式转换直接凭经验猜样式。优先使用：

- `tools/convert_word_to_docx.py`

如果模板、样文和默认规则冲突，必须列出冲突项并让用户选择。模板优先级高于样文，样文优先级高于默认规则。样式冲突细则见：

- `prompts/intake.md`
- `prompts/style_extractor.md`
- `references/default-style.md`

### 2. 冻结项目事实

先读项目代码和文档，再提炼固定的项目事实底稿。后续各章只能基于这份底稿扩写。

事实提取细则见：

- `prompts/fact_extractor.md`

### 3. 学模板和样文，不只学目录

如果用户提供模板或样文，必须优先进入“模板优先模式”，先学模板，再用样文补章节节奏。必须同时分析：

- 模板章节类型：封面、授权说明、中文摘要、英文摘要、目录、正文、参考文献、致谢、声明、附录、研究成果
- 结构：目录、页数、字数、图表节奏
- 模板：封面、目录页、摘要页、正文起始页、参考文献页、致谢页是否已存在
- 样式：标题、正文、摘要、关键词、图题表题、参考文献、致谢
- 细节：中英文正文的字体字号、段前段后、行距、首行缩进
- 标题：一级、二级、三级标题的字体字号、加粗、对齐方式、分页方式、段前段后
- 标题编号机制：标题文字是否已经写入 `第1章`、`1.1` 等编号；模板样式自身是否带自动编号；两者是否会叠加
- 模板特殊标题：`中文摘要`、`ABSTRACT`、`目　　录`、`参考文献`、`致　　谢`、`声　　明`、附录标题是否属于“非正文编号标题”
- 目录：是否已有自动目录、目录标题样式、目录层级
- 表图代码：图题、表题、表格内容、代码块或代码截图的插入位置与样式

如果模板和样文格式不一致，必须以模板为主、样文为辅。样文主要用于学习章节节奏和语言表达，模板主要用于决定最终交付样式。

对应资源：

- `prompts/sample_analyzer.md`
- `prompts/style_extractor.md`
- `prompts/language_style.md`
- `tools/analyze_docx_styles.py`
- `tools/analyze_sample_pdf.py`

### 3.5 先回传设计，再开写

模板和样文分析结束后，必须先把以下内容回传给用户确认：

1. 当前论文建议目录
2. 各章目标字数
3. 封面、目录、正文、标题、摘要、关键词、图题、表题、表格内容等版式样式
4. 与默认规则的冲突项
5. 模板中的“非正文编号标题”保留策略
6. 模板中的目录是否保留、重建还是替换
7. 标题编号由“模板样式自动生成”还是“正文文本显式写入”负责

必须明确等待用户确认目录和样式；若用户提出新的修改意见，以用户最后确认的版本为准，再进入正文写作。

### 4. 先定字数，再写作

写正文前必须生成目标章节字数表。默认优先贴近样文体量，不默认写厚。写完一章就统计一次，超出就压缩，明显不足就补充项目事实、实验分析和系统设计细节。

对应资源：

- `prompts/chapter_writer.md`
- `prompts/language_style.md`
- `tools/count_chapter_words.py`
- `references/chapter-patterns.md`

### 5. 图表与截图闭环

图表默认要求：

- 系统架构图
- E-R 图
- 关键流程图
- 数据表
- 测试用例表

如果存在 `mermaid` / `plantuml`，优先渲染为真实图片；若无法渲染，再退回源码或占位。

如果论文更偏科研型，需要概念图、机制示意图、方法框架图，则必须先做图示规划。优先使用：

- `prompts/diagram_designer.md`

如果环境中存在 `canvasdesign` 或等效绘图能力，优先用它生成科研型图示；如果没有，再退回 Mermaid / PlantUML。

对于论文中的公式：

- 如果正文中存在 LaTeX 公式，优先转成 Word 公式对象插入文档
- 如果环境不满足转换条件，再降级为普通文本，不允许默默丢失公式

如果存在 Chrome MCP、Playwright 或等效浏览器自动化能力，优先抓真实系统截图替换占位符。

图表生成必须额外检查：

- 图片宽度不得超过模板当前节的版心宽度
- 图片段落不得沿用正文固定行距
- 图题、表题必须为单倍行距
- Mermaid 无法渲染时，必须退回真实图片或程序化绘图，不允许把失败后的图示说明残留在图下

对应资源：

- `tools/render_mermaid.py`
- `tools/extract_screenshot_placeholders.py`

### 6. 参考文献先建池再回填

默认约束：

- 2020 年及以后
- 中文 10-12 篇
- 英文 3-5 篇
- 总数约 15 篇

必须优先真实可核验文献，不确定就不用。

对应资源：

- `prompts/reference_selector.md`
- `tools/build_reference_pool.py`

### 7. DOCX 成稿交付

如果环境具备 `doc` / `docx` 能力，必须生成 `.docx` 成稿，而不是只停留在 Markdown。

交付时必须优先走以下模式：

1. 用户给了模板：优先保留模板封面、目录、分页和样式，再把正文灌进去
2. 用户没给模板但给了样文：优先提取样文样式与章节节奏
3. 用户什么都没给：最后才退回默认样式规则

当输入包含“带完整示例内容的学校模板”时，必须额外执行以下规则：

1. 先区分前置部分、正文部分、后置部分、附录部分
2. `中文摘要`、`ABSTRACT`、`目　　录` 不得误套入正文章编号样式
3. `参考文献`、`致　　谢`、`声　　明`、`在学期间参加课题的研究成果` 不得显示为 `第N章`
4. 附录标题必须走模板附录样式，不得误套普通章标题
5. 如果模板标题样式自带编号，则正文源稿中的 `第1章`、`1.1`、`1.1.1` 等文本编号必须先剥离，避免叠加
6. 如果模板已经存在目录，默认优先复用或替换该目录位置，不允许额外再插入一个目录块
7. 模板原生特殊标题样式一旦识别成功，应优先保留模板原生格式，不得再用脚本默认字号和字体覆盖

表格生成规则：

1. 如果模板或样文中存在现成三线表，优先识别并克隆其结构
2. 再把论文表格内容灌进去
3. 如果没有可复用三线表，再退回程序生成的标准三线表

默认样式规则仅作为兜底：

- 摘要、Abstract、参考文献、致谢标题居中
- 摘要与 Abstract 独立分页
- 一级章节分页开始
- 中文正文宋体
- 英文正文 Times New Roman
- 中文关键词单独成段，顶格，“关键词：”标签使用黑体小四加粗，内容使用宋体小四
- 中英文摘要正文除关键词行外，默认首行缩进 2 字符
- 英文摘要正文使用 Times New Roman 小四
- 英文关键词单独成段，顶格，不首行缩进，使用 Times New Roman 小四
- 参考文献悬挂缩进

对应资源：

- `prompts/docx_formatter.md`
- `tools/generate_thesis_docx.py`
- `tools/convert_word_to_docx.py`
- `tools/analyze_docx_styles.py`
- `references/default-style.md`

### 8. 最终检查

交付前必须检查：

- 章节完整
- 字数接近样文目标
- 参考文献比例正确
- 图表编号连续
- 是否残留占位符
- `.docx` 是否真实存在
- 如果使用模板：封面是否保留、目录是否存在、目录是否可刷新、正文是否从正确位置开始
- 如果使用模板：摘要、Abstract、目录是否误显示为 `第N章`
- 如果使用模板：是否只保留一个目录
- 如果使用模板：正文标题是否存在编号叠加，如 `第1章第1章`、`1.11.1`
- 如果使用模板：`参考文献`、`致　　谢`、`声　　明`、附录是否误使用正文编号样式
- 如果使用模板：两个字的后置标题是否保持模板字间空格，如 `致　　谢`、`声　　明`
- 如果使用模板：后置标题是否被脚本错误覆盖为默认黑体/默认字号
- 如果输入源是 `.doc`：是否成功转换为 `.docx`
- 语言风格是否稳定、是否存在明显 AI 腔
- 逻辑链是否完整、结论是否被实验/测试支撑

最终检查细则见：

- `prompts/final_checker.md`
