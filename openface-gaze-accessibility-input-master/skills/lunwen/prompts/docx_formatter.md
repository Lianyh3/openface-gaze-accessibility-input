# DOCX Formatter Prompt

用于将论文源稿转成 `.docx`。

核心原则：

1. 如果用户给了模板，优先保留模板封面、目录、分页和样式
2. 如果用户给了样文，优先提取样文样式补充正文规则
3. 默认样式只做兜底，不允许默认规则覆盖用户模板

必须优先让生成脚本真实实现这些规则，而不是只在说明里写规则。

当输入包含模板时，生成流程应优先：

- 必要时先用 `tools/convert_word_to_docx.py` 把 `.doc` 转 `.docx`
- 用 `tools/analyze_docx_styles.py` 提取样式和正文替换起点
- 用 `tools/generate_thesis_docx.py --template ... --style-spec ... --replace-from ...` 生成最终文档

如果模板是“带完整示例内容的学校模板”，必须额外满足：

- `中文摘要`、`ABSTRACT`、`目　　录` 不得误生成 `第N章`
- `参考文献`、`致　　谢`、`声　　明`、附录、研究成果不得误生成 `第N章`
- 如果模板标题样式自带编号，必须先移除正文源稿中的文本编号，避免 `第1章第1章`、`1.11.1`
- 如果模板已存在目录位置，默认复用该位置，不允许额外再插入一个目录块
- 模板的 `章标题`、`章标题无大纲级别`、`附录标题` 等特殊样式，应优先保留模板原生字体字号和段落属性
- 图片宽度必须按当前节版心宽度自适应
- 图片段落不得沿用正文固定行距
- 图题、表题必须使用单倍行距
- Mermaid 渲染失败时，必须退回真实图片或程序化绘图，不允许把失败说明残留在图下

默认样式规则：

- 摘要、Abstract、参考文献、致谢标题居中
- 摘要与 Abstract 各自独立分页
- 一级标题分页开始
- 中文正文宋体
- 英文正文 Times New Roman
- 中文关键词单独成段，顶格，不首行缩进，“关键词：”标签黑体小四加粗，内容宋体小四
- 中英文摘要正文除关键词行外，保持正常首行缩进 2 字符
- 英文摘要正文 Times New Roman 小四
- 英文关键词单独成段，顶格，不首行缩进，`Keywords:` 标签加粗且为 Times New Roman 小四，内容为 Times New Roman 小四
- 图题、表题均使用单倍行距，段前段后为 0
- 参考文献悬挂缩进

如果存在截图或渲染图，优先嵌入真实图片。

如果没有模板但用户明确要求目录，生成脚本应创建可刷新的 Word 目录字段，而不是只输出纯文本“目录”。
