# Style Extractor Prompt

用于提取模板或样文版式样式。

核心原则：

- 模板优先于样文
- 样文优先于默认规则
- 不允许只看字体字号，必须同时看对齐、分页、段前段后、首行缩进和目录结构

必须关注：

- 模板是否为“空白模板”还是“带完整示例内容的模板”
- 模板章节分类：前置部分、正文、后置部分、附录
- 封面是否存在、封面表格是否存在、封面字段布局
- 目录页是否存在、目录标题样式、目录层级
- 一级、二级、三级、四级标题
- 标题编号是模板自动编号还是正文文本显式编号
- 标题文字中是否已写入 `第1章`、`1.1`、`1.1.1`
- `中文摘要`、`ABSTRACT`、`目　　录` 是否属于非正文章标题
- `参考文献`、`致谢/鸣谢`、`声明`、附录、研究成果是否属于后置标题
- 两字后置标题是否存在字间空格要求，如 `致　　谢`、`声　　明`
- 正文
- 摘要标题
- Abstract 标题
- 关键词格式
- 图题表题
- 表格内容字体字号
- 代码块或代码截图的标题与正文样式
- 参考文献
- 致谢 / 鸣谢
- 是否分页
- 是否居中或靠左
- 字体、字号、加粗
- 段前段后
- 行间距
- 首行缩进

如果用户给的是 `.docx` 模板或样文，优先使用：

- `tools/analyze_docx_styles.py`

提取出可复用的样式映射和正文起始标记。

如果提取出的样式与技能默认规则冲突，必须生成“冲突清单”并提示用户选择。

输出时必须把样式按模块列清楚，至少包括：

1. 封面保留策略
2. 目录保留策略
3. 前置部分、正文、后置部分、附录的边界
4. 一级标题样式
5. 二级标题样式
6. 三级标题样式
7. 四级标题样式
8. 中文正文样式
9. 英文正文样式
10. 摘要与 Abstract 样式
11. 关键词样式
12. 图题、表题、表格内容样式
13. 参考文献与致谢样式
14. 正文从模板哪个位置开始替换
15. 哪些标题必须保留模板原生样式
16. 标题编号由模板样式还是正文文本负责

样式分析完成后，必须与建议目录一起回传用户确认。
