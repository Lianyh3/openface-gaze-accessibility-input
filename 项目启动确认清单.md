## 7. 混合架构迁移计划（冻结）

### 7.1 迁移目标

1. 不破坏当前可演示链路（OpenFace + 事件流 + 候选增强）。
2. 将实时关键路径迁入 C++，降低延迟并提升稳定性。
3. 保留 Python 回退实现，保证实验可复现与可排障。

### 7.2 分阶段迁移范围

- [ ] Phase M0（接口冻结）：定义 `FrameFeatures/GazePoint/TargetEvent` 数据结构与测试样例。
- [ ] Phase M1（低风险模块迁移）：迁移 `normalizer + hit_test + dwell_detector` 到 C++。
- [ ] Phase M2（算法核心迁移）：迁移 `calibration + smoothing(EMA/OneEuro)` 到 C++。
- [ ] Phase M3（实时接入）：将 OpenFace 实时输出接入 C++ 核心并回传 Python 事件流。
- [ ] Phase M4（性能验收）：完成 Python 后端 vs C++ 后端对比报告（延迟/FPS/事件一致性）。

### 7.3 目录约定（预留）

1. `project/cpp_core/include/gaze_core/*.hpp`
2. `project/cpp_core/src/*.cpp`
3. `project/cpp_core/bindings/pybind_module.cpp`
4. `project/src/gaze_mvp/` 保留 orchestration 与 fallback 逻辑

### 7.4 验收标准（迁移完成判定）

1. 同一输入样例下，C++ 与 Python 事件序列一致率 >= 99%。
2. 关键路径端到端处理延迟较 Python 版本下降（目标 >= 20%）。
3. 任一 C++ 模块失败时，可通过配置切回 Python fallback，不影响演示链路。
